# KV未設定時の500エラー修正

## 変更内容

- `src/lib/kv.ts` に、`KV_REST_API_URL` と `KV_REST_API_TOKEN` の存在チェックを追加。
- 環境変数が未設定の場合、`@vercel/kv` ではなくプロセス内メモリ（`memoryEvent`）に保存するフォールバックを実装。
- 取得・保存時に `cloneEvent()` を通し、参照共有による意図しないミューテーションを防止。

## 意図（なぜ必要か）

- ローカル環境でKV環境変数が未設定だと、`/api/seat?name=...` が500で失敗し、デバッグ不能だったため。
- 本番ではKVを継続利用しつつ、ローカルでは最低限の動作保証を行うため。

## 選択理由

- KISS: 既存API（`getEvent` / `setEvent`）の外部インターフェースを変えず、内部で分岐するだけに留めた。
- YAGNI: DB抽象化層や新規ストレージ追加は行わず、必要最小限のフォールバックのみ導入した。
- DRY: フォールバック判定を共通定数（`hasKvEnv`）に集約し、同一条件の重複実装を避けた。

## 動作確認結果（ローカル）

- `npm run build`: 成功
- `GET /`: 200
- `GET /admin`: 200
- `GET /api/seat`: 400（名前未指定の想定どおり）
- `GET /api/seat?name=Tanaka`: 404（データ未割当の想定どおり、500解消）

## 補足

- このフォールバックはサーバープロセス再起動でメモリ内容が消えます。
- 永続化が必要な環境では、従来どおりKV環境変数を設定してください。
